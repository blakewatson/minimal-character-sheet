**A. Create `vite.config.js`** at project root (per BUILD-01, D-03, D-04, D-05, D-06):

```javascript
import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
  plugins: [
    vue(),
    tailwindcss(),
  ],
  build: {
    outDir: 'dist',
    manifest: true,
    emptyOutDir: true,
    rollupOptions: {
      input: {
        app: 'js/app.js',
        dashboard: 'js/dashboard.js',
        print: 'js/print.js',
        styles: 'css/app.css',
      },
    },
  },
  publicDir: 'public',
});
```

Key config decisions:
- `outDir: 'dist'` per D-06
- `manifest: true` generates `dist/.vite/manifest.json` for the PHP helper (BUILD-02 prereq)
- `emptyOutDir: true` is safe because all static files come from `publicDir` (per D-11 reproducibility)
- `publicDir: 'public'` -- Vite copies contents of `public/` into `dist/` after build
- Four inputs: 3 JS entry points per D-03, 1 CSS entry point per D-04
- `@tailwindcss/vite` plugin per D-05 replaces `@tailwindcss/postcss`

**B. Create `public/` directory and populate with static CSS** (per D-10, D-11):

Move these files from `dist/` to `public/` (they have no source files in the repo):
- `dist/quill.bubble.css` -> `public/quill.bubble.css` (alternatively copy from `node_modules/quill/dist/quill.bubble.css`)
- `dist/baguetteBox.min.css` -> `public/baguetteBox.min.css`
- `dist/gutenberg.min.css` -> `public/gutenberg.min.css`
- `dist/spacing.css` -> `public/spacing.css`
- `dist/print.css` -> `public/print.css`
- `dist/flex.css` -> `public/flex.css`
- `dist/reset.min.css` -> `public/reset.min.css`

Do NOT move `dist/notyf.min.css` -- it will be bundled into app.css (see below).
Do NOT move `dist/app.css` -- it is a build output, not a static file.

**C. Add Notyf CSS import to `css/app.css`** (per D-09, BUILD-04):

Add this import BEFORE the existing `@import 'tailwindcss';` line at the top of `css/app.css`:
```css

```

The notyf CSS will now be bundled into the compiled app.css output. The separate `notyf.min.css` file and its `<link>` tag in header.html are no longer needed (template cleanup is in Plan 02).
