## Directory Purposes

**`classes/controllers/`:**
- Purpose: Handle HTTP requests, enforce authentication, render templates or return JSON
- Contains: `Authentication.php`, `Dashboard.php`, `Admin.php`
- Key files: `Dashboard.php` (sheet CRUD), `Authentication.php` (login/register/password reset/CSRF)

**`classes/models/`:**
- Purpose: Database access layer — F3 Active Record mappers
- Contains: `User.php`, `Sheet.php`, `Token.php`
- Key files: `Sheet.php` (JSON blob storage, slug generation), `User.php` (auth tokens)

**`templates/`:**
- Purpose: Server-rendered HTML using F3 template engine
- Contains: One `.html` file per page/view, plus `header.html` and `footer.html` partials
- Key files: `header.html` (asset loading, opens `<body>`), `footer.html` (closes `<body>`, loads page-specific JS), `sheet.html` (Vue mount point), `dashboard.html`

**`js/components/`:**
- Purpose: Vue 2 single-file components (`.vue`) for the character sheet SPA
- Contains: 26 components covering all character sheet sections
- Key files: `Sheet.vue` (root component, save logic), `Field.vue` (reusable form input), `QuillEditor.vue` (rich text), `Tabs.vue` (navigation)

**`js/components/SearchResults/`:**
- Purpose: Detail panel components for in-app D&D content search
- Contains: `SpellDetails.vue`, `BackgroundDetails.vue`, `FeatDetails.vue`

**`js/languages/`:**
- Purpose: JSON translation files keyed by English string
- Contains: `en.json`, `de.json`

**`js/lib/`:**
- Purpose: Vendored JS libraries not installed via npm
- Contains: `markdown-it.min.js`

**`dist/`:**
- Purpose: Compiled and versioned static assets served to browsers
- Contains: `app.js`, `dashboard.js`, `print.js`, `app.css`, vendor CSS files, `mix-manifest.json`
- Generated: Yes (by `npm run dev`/`prod`)
- Committed: Yes

**`data/`:**
- Purpose: SQLite database and related files
- Contains: `db.sqlite3` (main database), `sessions/` directory
- Generated: Yes (auto-created on first run)
- Committed: No (in `.gitignore`)

**`migrations/`:**
- Purpose: SQL migration scripts for schema changes
- Contains: `001.php` through `004.php`, `sheets.php`, `users.php`

**`etl/`:**
- Purpose: One-off data import scripts for D&D 5e content (spells, feats, backgrounds)
- Contains: Import scripts; processed data goes into `data/`

## Key File Locations

**Entry Points:**
- `index.php`: PHP application bootstrap and all route definitions
- `js/app.js`: Character sheet SPA — Vue setup, mounts `Sheet.vue`
- `js/dashboard.js`: Dashboard JS — binds public-toggle and delete interactions
- `js/print.js`: Print view — mounts `Print.vue`

**Configuration:**
- `webpack.mix.js`: Build configuration — defines JS/CSS compilation and versioning
- `composer.json`: PHP dependencies (Fat-Free Framework, Dotenv, Postmark SDK)
- `package.json`: Node dependencies (Vue 2, Vuex, Laravel Mix, Tailwind CSS v4)
- `.env`: Required env vars (`POSTMARK_SECRET`, optional `ENV`, `DEBUG`)

**Core Logic:**
- `js/store.js`: Entire Vuex store — state shape, all mutations, hydration actions
- `js/i18n.js`: Translation system (`t()` function) and Vue plugin
- `js/utils.js`: `throttle()`, `signedNumString()`, `copyHtmlToClipboard()`
- `js/mixins.js`: `listMixin` for growable list fields
- `js/settings.js`: localStorage-based settings (dark mode, textarea font, language) — runs as IIFE on every page

**Templates:**
- `templates/header.html`: HTML `<head>`, opens `<body>` — included in every page
- `templates/footer.html`: Closes `<body>`, loads per-page JS bundles — included in every page
- `templates/sheet.html`: `<div id="sheet">` Vue mount point and hidden CSRF/slug inputs
- `templates/dashboard.html`: Sheet list, settings controls, CSRF input

**Database:**
- `data/db.sqlite3`: SQLite database (tables: `user`, `sheet`, `sessions`)

## Naming Conventions

**Files:**
- PHP controllers: PascalCase — `Dashboard.php`, `Authentication.php`
- PHP models: PascalCase — `Sheet.php`, `User.php`
- Vue components: PascalCase — `Sheet.vue`, `QuillEditor.vue`, `TrackableFields.vue`
- JS modules: camelCase — `store.js`, `mixins.js`, `levelData.js` (exception: `level-data.js`)
- Templates: kebab-case — `add-sheet.html`, `password-reset.html`
- CSS: kebab-case — `app.css`

**Directories:**
- PHP: lowercase — `classes/controllers/`, `classes/models/`
- JS: camelCase/lowercase — `components/`, `languages/`, `lib/`
- Templates: flat (no subdirectories)

## Where to Add New Code

**New character sheet section (Vue component):**
- Implementation: `js/components/YourSection.vue`
- Register in: `js/components/Sheet.vue` components object
- State fields: Add to `js/store.js` state, add mutations for each field
- Translations: Add keys to `js/languages/en.json` and `js/languages/de.json`

**New backend route:**
- Route definition: `index.php` (add `$f3->route(...)` call)
- Handler: Add method to the appropriate controller in `classes/controllers/`
- Template (if needed): Add `.html` file to `templates/`

**New PHP model:**
- Implementation: `classes/models/YourModel.php` (extend `\DB\SQL\Mapper`)
- Auto-loaded by F3 via `AUTOLOAD` setting in `index.php`

**New utility function:**
- Shared JS helper: `js/utils.js` (export named function)
- Shared Vue mixin: `js/mixins.js` (export named mixin object)

**New translation string:**
- Add English key/value to `js/languages/en.json`
- Add German translation to `js/languages/de.json`
- Use in component with `this.$t('Your string')` or `t('Your string')` in non-component JS

**Database schema change:**
- Add migration script: `migrations/005.php` (incrementing number)
