<objective>
Fix the infinite rebuild loop in `npm run watch` caused by `emptyOutDir: true` in vite.config.js.

When Vite runs in watch mode, `emptyOutDir: true` deletes the `dist/` directory at the start of each build, which triggers the file watcher to detect changes and rebuild, creating an infinite loop.

Fix: conditionally disable `emptyOutDir` when running in watch mode by checking `process.env.ROLLUP_WATCH`, which Vite/Rollup sets to `"true"` during watch builds.
</objective>

<context>
@vite.config.js
</context>

<tasks>

<task type="auto">
  <name>Task 1: Conditionally disable emptyOutDir in watch mode</name>
  <files>vite.config.js</files>
  <action>
    In vite.config.js, change the `emptyOutDir` value from the hardcoded `true` to a conditional expression:

    ```js
    emptyOutDir: !process.env.ROLLUP_WATCH,
    ```

    This keeps `emptyOutDir: true` for normal builds (`npm run dev`, `npm run prod`) but sets it to `false` during watch mode (`npm run watch`), breaking the infinite loop.

    No other changes needed. Do not modify any other config options.
  </action>
  <verify>
    <automated>cd /workspace && npx vite build --watch --mode development 2>&1 | head -20</automated>
    Verify: build completes once and waits for changes (does not loop). Kill after confirming stable.
  </verify>
  <done>
    `npm run watch` completes an initial build and waits for file changes without looping. Normal `npm run dev` and `npm run prod` still clear dist/ before building.
  </done>
</task>
